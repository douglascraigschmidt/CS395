Movies Microservice (Assignment 1b)

The purpose of the second part of this two-part assignment is to acquaint you with developing synchronous microservices with Spring WebMVC, Java structured concurrency, and Java regular expressions. In this assignment you'll evolve the synchronous microservice that used Spring WebMVC from assignment 1a to use the StructuredTaskScope.ShutdownOnFailure class to concurrently return Lists of objects containing information about movies from a movie recommender system we'll use throughout this course.

The code you'll write for this assignment does not have a GUI and runs as an Spring application with an automated test program. Please feel free to use this test program to help debug your solution.

Assignment Structure and Functionality

The Movies microservice is packaged as a project using IntelliJ. This app is written in Java and demonstrates various Spring WebMVC and Java structured concurrency features. For the purposes of Assignment 1b, however, you only need to be aware of the following directories:

  • recommender/movies, which contains the skeleton(s) you'll need to fill in, as described below.
  • movies/src/test, which is a set of unit tests that exercise the features you're implementing to evaluate whether you've correctly satisfied this assignment's requirements.

To compile this code you need to use the provided IntelliJ project. You can run this project by clicking the green arrow next to the "MoviesTest" dropbox menu in the IntelliJ IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" markers)!!! In particular, you'll need to finish implementing the "TODO" tasks for this assignment in the following files and folders:

  • MoviesController.java. You'll need to complete the "TODO" tasks in various fields/methods to define endpoint handler methods that return Lists of objects containing information about movies.
  • MoviesService.java. You'll need to complete the "TODO" tasks in various fields/methods called by the MoviesController that use Java structured concurrency and Java sequential streams to return Lists of objects containing information about movies. You'll also use Java regular expressions to get more precise control over searches.
  • ServerBeans.java. You'll need to complete the "TODO" tasks in factory methods (known as "Beans") that can be injected into Java classes using the Spring Autowired annotation to load a dataset containing movie titles and descriptive vectors.
  • MoviesSyncProxy.java. You'll need to complete the "TODO" tasks to implement proxies to various endpoints in the Movies microservice. You'll also use Java regular expressions to get more precise control over searches.

Again, your solution should strive to use no loops or if/else statements in this assignment and instead use Java functional programming features (including lambda expressions, method references, and sequential stream aggregate operations) and Java structured concurrency mechanisms. Your solution will be considered correct if it passes all the unit tests, is commented thoroughly, and addresses my review comments.
