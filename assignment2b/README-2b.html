Database Microservice (Assignment 2b)

The purpose of the second part of this assignment is to deepen your understanding of how to develop multiple synchronous microservices with Spring WebMVC, the Eureka Discovery Service, the Spring Cloud Gateway, the Retrofit type-safe client API, and the Jakarta Persistence API (JPA). In this assignment you'll apply JPA to create a database that stores information about movies and supports various SQL queries (including a custom SQL query), as well as apply Retrofit to create several type-safe client APIs that access database and timer microservices via an API gateway.

The code you'll write for this assignment does not have a GUI and runs as an Spring application with an integration main program. Please feel free to use this main program to help debug your solution.

  • A tutorial on Retrofit appears here.
  • I also recommend you use ChatGPT to help learn and code these various APIs.

Assignment Structure and Functionality

The Database microservice is packaged as a project using IntelliJ. This app is written in Java and demonstrates various Spring WebMVC and Java concurrency and persistence features. For the purposes of Assignment 2b, however, just focus on the following directories:

  • client, which contains a client main driver program containing proxy skeleton(s) you'll need to fill in, as described below.
  • database, which contains the database microservice server skeleton(s) you'll need to fill in, as also described below.
You'll also need to copy the synchronous movies-ex microservice implementation from assignment 2a, which is found in these folders:
  • movies-ex, which contains the skeleton(s) you'll need to fill in, as described below.
  • movies-ex/src/test, which is a set of unit tests that exercise the features you're implementing to evaluate whether you've correctly satisfied this earlier assignment's requirements.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" markers or your submission will not be reviewed)!!! In particular, you'll need to finish implementing the "TODO" tasks for this assignment in the following files and folders:

  • client, where you'll need to complete the "TODO" tasks in various fields/methods to implement a type-safe client using Retrofit.
  • database, where you'll need to complete the "TODO" tasks in various fields/methods of the MultiQueryRepositoryImpl.java and DatabaseService.java files that use declarative and programmatic features of the Jakarta Persistence API together with Java parallel streams to return lists of objects containing information about movies.
  • You'll also need to complete the "TODO" tasks in factory methods in the ServerBeans.java and ClientBeans.java files that can be injected into Java classes using the Spring Autowired annotation.
  • Finally, you'll need to implement the "TODO" tasks in the Movie.java file.

As usual, your solution should strive to use few (if any) loops or if/else statements in this assignment and instead use Java functional programming features (including lambda expressions, method references, and parallel stream aggregate operations). Your solution will be considered correct if it passes all the unit tests, is commented thoroughly, and addresses my review comments. Solutions that are not commented thoroughly will not be reviewed.

Unit Tests

There are unit tests that exercise all the "TODO" tasks you must complete. To compile and run these tests you need to use the provided IntelliJ project. You can run all the unit tests by opening the Gradle tab, followed by navigating down to the "Tasks > verification > check" submenu. All you need to do is double click on the "check" submenu and all the tests will build and run automatically.

These tests increase our collective confidence that your solution is working as expected. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the tests to detect problems in your code. In particular, I will also provide videos that walk through "frequently made mistakes" that you should address. Please address these issues prior to your submission(s) to streamline our programming assignment review and assessment process.
